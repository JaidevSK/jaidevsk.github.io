Welcome to Algolang!

This documentation serves as a comprehensive guide to understanding and utilizing Algolang. Whether you're a complete beginner or an experienced developer, this guide aims to provide you with the necessary tools and knowledge to effectively write and execute programs.

Our language is designed with simplicity in mind, allowing you to focus on solving problems rather than getting stuck in complex syntax. The left sidebar contains an index of topics, each covering a specific aspect of the language. We encourage you to explore each section in order to gain a thorough understanding of the language's capabilities.

Getting Started

Before you can start writing code, you need to set up your development environment. This involves installing the language interpreter or compiler and configuring your text editor or IDE.

  1. Installation:

    The compiler has not been released. It will be updated as soon as the Course is completed.

  2. Running your first program:

    First create a file in text editor with .algl extension, write a simple "Hello, World!" program, and execute it using the interpreter or compiler.

    print "Hello, World!";
    Run it with the command 
    sh run.sh file_name
    for using it with the Compiler. Run it with the command 
    python3 RUN.py file_name
    for using it with the Interpreter.

    This will print Hello World! in the terminal.

  3. User Inputs

    4. For taking user inputs, use the input function.

    let name = input("Enter your name: ");
print "Hello, " + name + "!";
  4. Comments

    5. Comments can be added using # symbol.

    # This is a comment
print "Hello, World!"; # This is another comment

Variables and Datatypes

Variables

Variables are used to store data that can be manipulated during program execution. You declare variables using the let keyword, followed by the variable name and an optional initial value.

let x = 10;
let message = "Welcome!";
let isReady; # Declaration without initialization

Variable names must start with a letter or underscore and can contain letters, numbers, and underscores. Variable Scoping is defined in the page on Typing and Scoping.

Supported Datatypes

Integers

Our language supports integer numbers, which are numbers without decimal points. You can perform arithmetic operations on numbers using operators like +, -, *, /, //, %, and ^.

Floats

Floats are numbers with decimal points. They can be used to represent fractional values and perform operations like addition, subtraction, multiplication, and division.

Strings

Strings are sequences of characters enclosed in single or double quotes. The string should support either single or double quotes. There are come more string operations supported like concatenation, indexing, discussed in the Data Structures page. nl is a special character for newline.

Booleans

Booleans represent true or false values. They are used in conditional statements and logical operations.

Compound Data Types

let num1 = 20;
let num2 = 5;
let sum = num1 + num2;
print sum;

Operators

Operators are symbols that perform operations on values. They are used to manipulate data and control the flow of execution.

Arithmetic Operators

  • + (Addition): Adds two numbers.
  • - (Subtraction): Subtracts one number from another.
  • * (Multiplication): Multiplies two numbers.
  • / (Division): Divides one number by another.
  • // (Integer Division): Divides and rounds down to the nearest whole number.
  • % (Modulo): Returns the remainder of a division.
  • ^ (Exponentiation): Raises a number to a power.

Comparison Operators

  • == (Equal): Checks if two values are equal.
  • != (Not Equal): Checks if two values are not equal.
  • < (Less Than): Checks if the left value is less than the right value.
  • > (Greater Than): Checks if the left value is greater than the right value.
  • <= (Less Than or Equal): Checks if the left value is less than or equal to the right value.
  • >= (Greater Than or Equal): Checks if the left value is greater than or equal to the right value.

Logical Operators

  • and: Returns true if both conditions are true.
  • or: Returns true if at least one condition is true.
  • not: Reverses a true or false value.

Control Flow

Control flow statements allow you to control the order in which code is executed based on certain conditions or loops.

`if` Statements

The if statement executes a block of code only if a specified condition is true. You can also use else and elseif to handle alternative conditions.

let num = 10;
if num > 0 {
    print "POSITIVE";
}
elif num == 0 {
    print "ZERO";
} 
else {
    print "NEGATIVE";
}

# Prints POSITIVE

Here if-elif-else check for the conditions and if they are satisfied, the corresponding block is executed.

`while` Loops

The while loop repeats a block of code as long as a specified condition is true.

let i = 0;
while i < 5 {
    print i;
    i = i + 1;
}

The while loop has th

`breakloop` and `skipit`

breakloop immediately exits a loop, while skipit skips the rest of the current iteration and moves to the next.

let i = 0;
while i < 10 {
    i = i + 1;
    if i == 5 {
        skipit;
    }
    if i == 8 {
        breakloop;
    }
    print i;
}

The above code runs from 0 to 10, but skips 5 and breaks at 8.

Functions

Functions are reusable blocks of code that perform specific tasks. They help organize code and improve readability.

You define functions using the alg keyword, followed by the function name, a list of parameters in parentheses, and the function body enclosed in curly braces.

let add = alg(a, b) {
            return a + b;
        };
        let result = add(3, 5);
        print result;

Functions can also be assigned to variables and used as values, demonstrating first-class function capabilities.

let myFunc = add;
let anotherResult = myFunc(10, 20);
print anotherResult;

When interpreted, the language supports closures, allowing functions to capture variables from their surrounding scope.

let outer = alg(x) {
    let inner = alg(y) {
        return x + y;
    };
    return inner;
};

let closureFunc = outer(10);
print closureFunc(5);

Supported Operations

  • Declaration
    • let add = alg(a, b) {
    return a + b;
};
  • Function Call
    • let result = add(3, 5);
print result; # Output: 8
  • Assignment
    • let myFunc = add;
let anotherResult = myFunc(10, 20);
print anotherResult; # Output: 30
  • Closures
    • let outer = alg(x) {
    let inner = alg(y) {
        return x + y;
    };
    return inner;
};

let closureFunc = outer(10);
print closureFunc(5); # Output: 15
  • Recursion
    • let factorial = alg(n) {
    if n == 0 {
        return 1;
    }
    return n * factorial(n - 1);
};

let result = factorial(5);
print result; # Output: 120
  • Novel Feature: Functional Addition

    • Our language supports functional addition, allowing you to combine functions using the + operator.

    let fn1 = alg(a) {
    return a + 1;
};
let fn2 = alg(a) {
    return a + 2;
};
let fn3 = fn1 + fn2;
print fn3(5); # Output: 8
  • Novel Feature: Function Multiplication

    • Our language supports function multiplication, allowing you to compose functions using the * operator.

    let fn1 = alg(a) {
    return a + 1;
};
let fn2 = fn1*5;

print fn2(5); # Output: 10

    Data Structures

    Data structures are used to organize and store collections of data.

    Strings

    Strings are sequences of characters enclosed in single or double quotes. You can perform various operations on strings, such as concatenation and slicing.

    Supported Operations

  • Initialization
    • let message = "Hello, World!";
  • Concatenation: Combine two strings into one.
    • let greeting = "Hello, ";
let world = "World!";
let message = greeting + world;
print message; # Output: Hello, World!
  • Length: Get the number of characters in a string.
    • let message = "Hello, World!";
let length = len(message);
print length; # Output: 13
  • Indexing: Access characters by index (assignment is not allowed in interpreted).
    • let message = "Hello, World!";
print message[7]; # Output: W

    Lists and Vectors

    Lists are ordered collections of items. You can access elements by their index, starting from 0.

    let myList = {1, 2, 3, 4, 5}; # Create a list
print myList[0]; # Output: 1
myList[2] = 10; # Modify an element
print myList;

    Supported Operations

  • Declaration
    • let myList = {1, 2, 3, 4, 5};
  • Indexing: Access elements by index.
    • let myList = {1, 2, 3, 4, 5};
print myList[2]; # Output: 3
  • Assignment: Modify elements by index.
    • let myList = {1, 2, 3, 4, 5};
myList[2] = 10;
print myList; # Output: {1, 2, 10, 4, 5}
  • Concatenation: Combine two lists into one.
    • let list1 = {1, 2, 3};
let list2 = {4, 5, 6};
let combined = list1 + list2;
print combined; # Output: {1, 2, 3, 4, 5, 6}

    Vectors

    Vectors (ndarrays) are one-dimensional arrays that can store multiple values of the same type. They are indexed by integers starting from 0. In the compiled version, they are treated as lists, while in the interpreted version, they are treated as numpy arrays.

    let myVector = [1, 2, 3, 4, 5]; # Create a vector
print myVector[0]; # Output: 1
myVector[2] = 10; # Modify an element
print myVector;

    Supported Operations

  • All the operations supported by Lists
  • + Operation
    • let myVector1 = [1, 2, 3];
let myVector2 = [4, 5, 6];
let combinedVector = myVector1 + myVector2;
print combinedVector; # Output: [5, 7, 9]
  • Similarly, it supports -, *, / and ** pointwise similarly to integers in interpreted version.
  • Length
    • let myVector = [1, 2, 3, 4, 5];
let length = len(myVector);
print length; # Output: 5

    Dictionaries (Hash Tables)

    Dictionaries store data as key-value pairs. Keys must be unique and immutable.

    dict myDict;
myDict["name"] = "NAME";
myDict["age"] = 100;
print myDict["name"];
print myDict["age"];
print myDict; # In the interpreted version, it will be printed as a dictionary.

    Supported Operations

  • Declaration
    • dict myDict;
  • Assignment and Retrieval: Add or modify key-value pairs.
    • dict myDict;
myDict["name"] = "NAME";
myDict["age"] = 100;
print myDict["name"];
print myDict["age"];
  • Keys: Get all keys or values from the dictionary.
    • dict myDict;
myDict["name"] = "NAME";
myDict["age"] = 100;
keys = keys(myDict);
print keys; # Output: ["name", "age"]

    Typing and Scoping

    Dynamic Typing

    Our language uses dynamic typing, meaning the type of a variable is determined at runtime. You don't need to explicitly declare the type of a variable.

    let x = 10; # x is an integer
x = "Hello"; # x is now a string
print x;

    Dynamic/Lexical Scoping

    When the code is interpreted, the scoping is dynamic. When the code is compiled, the scoping is lexical.

    Dynamic Scoping: Variables are looked up in the scope where the function is called.

    let x = 10;
let myFunc = alg() {
    print x;
};
let outer = alg() {
    let x = 20;
    myFunc();
};
outer(); # Output: 10 (dynamic scoping)

    Lexical Scoping: Variables are looked up in the scope where the function is defined.

    let x = 10;
let myFunc = alg() {
    print x;
};
let outer = alg() {
    let x = 20;
    myFunc();
};
outer(); # Output: 20 (lexical scoping when compiled)

    Advanced Features

    Our language offers several advanced features to enhance your programming experience.

    Run-Time Type Checking

    The interpreter performs run-time type checking to ensure that operations are performed on compatible data types. If a type mismatch occurs, an error is raised.

    Extendable Interpreter Functionality (SysCalls)

    You can extend the interpreter with user-defined SysCalls to add custom functionality.

    AST Visualization (Graphviz)

    The Abstract Syntax Tree (AST) of your code can be visualized using Graphviz, helping you understand the structure of your program.

    Basic Plotting (Termplotlib)

    You can create simple plots in the terminal using Termplotlib.

    Syntax Highlighting

    Our language supports syntax highlighting in the terminal using escape sequences and in VS Code using the AlgSyn extension, making your code easier to read.

    Compilation to MIPS32 (Virtual Machine)

    Your code can be compiled to MIPS32 assembly and run on our custom virtual machine.

    Key Design Decisions and Innovations

    Our language was designed with simplicity and innovation in mind. Here are some key design decisions that influenced the development of Algolang.

    • Dynamic Typing: We chose dynamic typing to allow flexibility in variable declarations and assignments without the need for explicit type annotations.
    • Dynamic/Lexical Scoping: We implemented dynamic scoping when interpreted and lexical scoping when compiled to provide a balance between flexibility and predictability.
    • Lists and Vectors: We introduced lists and vectors to support collections of data and provide a convenient way to store and manipulate values.
    • Run-Time Type Checking: We included run-time type checking to catch type errors early and ensure safe and consistent behavior.
    • Extendable Interpreter Functionality: We enabled users to extend the interpreter with custom SysCalls to add new features and capabilities.
    • Functions as Values: We treated functions as first-class values, allowing them to be assigned to variables, passed as arguments, and returned from other functions.
    • Function Operations: We added additional operations like function addition and multiplication to provide more flexibility and composability.
    • Closure of Functions: We supported closures in the interpreted version to capture variables from the surrounding scope and create self-contained functions.
    • AST Visualization: We implemented AST visualization using Graphviz to help users understand the structure of their code and debug more effectively.
    • Basic Plotting: We included basic plotting using Termplotlib to allow users to visualize data and results directly in the terminal.
    • Syntax Highlighting: We provided syntax highlighting in the terminal using escape sequences and in VS Code using the AlgSyn extension to improve code readability and aesthetics.
    • Compilation to MIPS32: We compiled code to MIPS32 assembly to run on our custom virtual machine, enabling efficient execution and performance.
    • VM Design: Register based VM with a Stack for the language like MIPS32

    Future Development

    • Quantum VM: We want to explore the possibility of compiling Algolang to a quantum virtual machine to leverage quantum computing capabilities as well as support for quantum gate level operations.
    • Two Modes of Operation: Quantum + Classical mode of operation will be supported by Python based VM, while the C based VM, focussed on performance, will support only classical operations.
    • Development of a Coding Application: We plan to develop a coding application that integrates Algolang with a user-friendly interface, code editor, and the interpreter/compiler to enhance the development experience.
    • More libraries: We plan to add more libraries to support advanced data structures, algorithms, and mathematical operations.
    • Optimization: We aim to optimize the interpreter and compiler to improve performance and reduce execution time.
